Google Drive 連携

Google Cloud で OAuth クライアントを作成し、ここで認証情報とスキャン対象フォルダを保存します。 設定は {{ data_dir }}/runtime_roots.json および OAuth 用 JSON に保存されます。

1. Google Cloud で OAuth クライアントを作る

SnapStack は API キーではなく OAuth 2.0 クライアント ID / シークレット を使います（Drive のファイル一覧・読み取り用）。

  1. Google Cloud Console でプロジェクトを作成または選択する。
  2. API とサービス → ライブラリ →「Google Drive API」を検索して 有効にする。
  3. API とサービス → OAuth 同意画面 を開き、ユーザータイプ（外部で可）とアプリ名・連絡先を入力して保存する。
  4. 同じ画面の テストユーザー に、ログインに使う Google アカウントを追加する（公開前は必須）。
  5. API とサービス → 認証情報 → 認証情報を作成 → OAuth クライアント ID。
  6. アプリケーションの種類: ウェブアプリケーション。
    承認済みのリダイレクト URI に次を追加（画面下部の「現在のコールバック URL」と一致させる）: {{ default_redirect_uri }}
  7. 作成後に表示される クライアント ID と クライアント シークレット を、下のフォームに貼り付けて保存する。

2. OAuth クライアント（SnapStack に保存）

保存先: {{ data_dir }}/google_oauth_client.json。 環境変数 SNAPSTACK_GOOGLE_* がある場合はそちらが優先されます。

3. スキャン対象フォルダ（Drive）

有効にすると、トップ画面の「対象フォルダ」に Google Drive が表示されます（Google への接続も必要です）。
